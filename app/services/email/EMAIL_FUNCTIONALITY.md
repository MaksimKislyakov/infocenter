# Функциональность отправки паролей на почту

## Описание

При создании пользователя администратором система автоматически:
1. **Генерирует безопасный пароль** (случайный, неизвестный админу)
2. **Отправляет пароль ТОЛЬКО на email пользователя** 
3. **Хранит пароль в БД как хешированный** (bcrypt)

**Критически важно:** Администратор **НИКОГДА** не может узнать пароль пользователя. Пароль известен только самому пользователю.

## Конфигурация

Добавьте в `.env` файл SMTP параметры (уже добавлены):

```env
# Почта для отправки
SMPT_MAIL=example@mail.ru
SMTP_MAIL_PASSWORD=your_password
```

**Примечание:** Текущие параметры настроены для mail.ru. Если меняете провайдера, обновите:
- `SMPT_MAIL` - адрес от которого отправляются письма
- `SMTP_MAIL_PASSWORD` - пароль приложения (не основной пароль!)
- В `EmailService` может потребоваться изменить SMTP сервер и порт (текущие: smtp.mail.ru, 465)

## API Эндпоинты

### 1. Создание пользователя (только админ)

```bash
POST /users/
Content-Type: application/json
Authorization: Bearer <admin_token>

{
  "login": "john_doe",
  "full_name": "John Doe",
  "role": "user",
  "email": "john@example.com"
}
```

**Результат:** 
- Пароль автоматически сгенерирован
- Отправлен на `john@example.com`
- **Администратор его не видит**

**Ответ:** Возвращается объект пользователя **БЕЗ пароля**

### 2. Смена пароля пользователем (требует текущий пароль)

```bash
PATCH /users/me/password
Content-Type: application/json
Authorization: Bearer <user_token>

{
  "current_password": "OldPassword123!",
  "new_password": "NewPassword456!"
}
```

**Результат:** Пароль успешно изменён

### 3. Сброс пароля админом (если пользователь потерял пароль)

```bash
POST /users/{user_id}/reset-password
Authorization: Bearer <admin_token>
```

**Что происходит:**
1. Генерируется новый случайный пароль
2. Отправляется на email пользователя
3. Старый пароль становится недействительным
4. **Администратор пароль не видит**

**Результат ответа:** `{"detail": "Password reset successfully. New password sent to user's email."}`

## Сценарии использования

### Сценарий 1: Создание нового пользователя (нормальный процесс)

```
1. Админ создаёт пользователя через POST /users/
   - Указывает login, full_name, role, email
   - НЕ указывает пароль

2. Система:
   - Генерирует пароль (12 символов, с буквами, цифрами, спецсимволами)
   - Отправляет его на email

3. Пользователь:
   - Получает письмо с паролем
   - Логинится в систему
   - При необходимости меняет пароль на свой
```

### Сценарий 2: Пользователь забыл пароль

```
1. Пользователь:
   - Не может вспомнить пароль
   - Обращается к администратору

2. Админ:
   - Вызывает POST /users/{user_id}/reset-password

3. Система:
   - Генерирует новый пароль
   - Отправляет на email пользователя
   - Старый пароль больше не работает

4. Пользователь:
   - Логинится с новым паролем
   - Меняет его на свой через PATCH /users/me/password
```

### Сценарий 3: Пользователь меняет пароль

```
1. Пользователь:
   - Вызывает PATCH /users/me/password
   - Передаёт текущий пароль и новый

2. Система:
   - Проверяет текущий пароль
   - Если верен - меняет на новый
   - Если неверен - возвращает ошибку 400
```

## Технические детали

### EmailService (`app/services/email_service.py`)

Сервис для отправки писем по SMTP:
- Автоматически обрабатывает ошибки SMTP
- Логирует результаты отправки
- Использует HTML и TEXT версии письма
- Пропускает отправку если SMTP не сконфигурирован

### Генерация пароля (`app/services/auth_service.py`)

Функция `generate_password(length=12)`:
- Длина по умолчанию: 12 символов
- **Содержит обязательно:**
  - минимум 1 строчную букву (a-z)
  - минимум 1 заглавную букву (A-Z)
  - минимум 1 цифру (0-9)
  - минимум 1 спецсимвол (!@#$%^&*)
- Использует `secrets` для криптографически безопасной генерации

### Схема UserCreate

Пароль теперь **не передаётся в API**:
```python
password: Optional[str] = Field(None, ...)
```

При создании через API пароль игнорируется и всегда генерируется автоматически.

## Логирование

Все события отправки писем логируются:

```
INFO: Password email sent successfully to john@example.com
WARNING: Failed to send password email to john@example.com
ERROR: SMTP authentication failed: ...
```

Проверяйте логи в случае проблем с отправкой писем.

## Безопасность

✅ **Администратор не может:**
- Узнать пароль пользователя при создании
- Узнать пароль пользователя при сбросе
- Установить пароль явно

✅ **Пользователь может:**
- Менять свой пароль через PATCH /users/me/password
- Требуется ввести текущий пароль для подтверждения

✅ **При потере пароля:**
- Администратор может сбросить через POST /users/{user_id}/reset-password
- Новый пароль отправляется только на email
- Администратор пароль не видит
